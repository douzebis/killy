# 0009 — First-boot host key enrollment

- **Status:** implemented
- **Implemented in:** 2026-03-13

---

## Background

Spec 0005 has `killy-install` pre-generate the target system's SSH host key
and derive the host age key from it, then run `sops updatekeys` to add it as
a recipient in `install-config.yaml`. This approach has two problems:

1. The SSH host key is generated in the installer environment rather than by
   the target system itself, which is non-standard and slightly less clean.

2. The build-host `.sops.yaml` must carry killy's host age key as a recipient
   so that `sops updatekeys` can find and replace it on reinstall. This key
   goes stale on each reinstall. Spec 0008 removed it from `.sops.yaml`, which
   broke `killy-install`'s regex-based key replacement.

The cleaner approach: let the target system generate its own SSH host keys on
first boot (standard NixOS behaviour), then have a one-time first-boot service
use the Yubikey to decrypt secrets and enroll the host key as a recipient.

---

## Goals

1. `killy-install` no longer pre-generates SSH host keys or runs `sops updatekeys`.
2. On first boot, a one-time systemd service:
   a. Waits for the SSH host key to be generated by the standard NixOS activation.
   b. Uses the Yubikey (via `yk-unwrap`) to obtain the install age key.
   c. Derives the host age public key from the SSH host key via `ssh-to-age`.
   d. Adds the host age key as a recipient in `/etc/nixos/.sops.yaml`.
   e. Runs `sops updatekeys` on `/etc/nixos/install-config.yaml`.
   f. Triggers sops-nix to re-activate so secrets are decrypted with the host key.
   g. Marks itself as done (drop a stamp file) so it never runs again.
3. On subsequent boots, sops-nix uses the SSH host key directly as usual.
4. The Yubikey must be present on first boot (same requirement as the installer).
5. `killy-install` is simplified — no SSH host key generation, no `sops updatekeys`.

---

## Non-goals

- Fully unattended first boot without the Yubikey.
- Rotating the host key after enrollment.
- Supporting multiple host keys.

---

## Specification

### 9.1 Changes to `killy-install`

Remove steps 6 (SSH host key generation) and 7 (host age key derivation +
`sops updatekeys`) from `killy-install`. The install script still:

- Copies `killy/` to `/mnt/etc/nixos/`
- Copies the live `install-config.yaml` (encrypted for operator + Yubikey keys)
  to `/mnt/etc/nixos/install-config.yaml`
- Runs `nixos-install`

`install-config.yaml` on the installed system is encrypted for operator +
Yubikey keys only. The host key is added on first boot.

### 9.2 Enrollment service (`killy-enroll-host-key`)

A systemd service added to `killy/modules/base.nix` that runs on every boot
but is a no-op once the host key is already enrolled:

```
killy-enroll-host-key.service
  After=  sshd-keygen.service
  Before= sops-install-secrets.service
  Wants=  sops-install-secrets.service
  Type=oneshot
  RemainAfterExit=yes
```

No stamp file. Instead the service checks whether `install-config.yaml` can
already be decrypted with the SSH host key:

```bash
sops decrypt /etc/nixos/install-config.yaml > /dev/null 2>&1
```

If this succeeds, the host key is already a recipient — exit immediately.
If it fails, proceed with enrollment. This makes the service idempotent and
self-healing.

**Enrollment steps (only when host key is not yet a recipient):**

1. Run `yk-unwrap` to obtain the install age key, writing it to
   `/run/age-install-key`. Retry loop until Yubikey is present.
2. Derive host age public key:
   ```bash
   ssh-to-age -i /etc/ssh/ssh_host_ed25519_key.pub
   ```
3. Add host age key to `/etc/nixos/.sops.yaml` (insert under a
   `# killy host key` comment if absent, replace if present).
4. Run `sops updatekeys --yes /etc/nixos/install-config.yaml` with
   `SOPS_AGE_KEY=<install_age_key>`.

### 9.3 Yubikey tooling on the installed system

Add to `killy/modules/base.nix` environment.systemPackages:
- `yubikey-manager` (for `ykman`)
- `pcscd` enabled as a service
- `age` + `ssh-to-age`
- The `yk-unwrap.py` script (copied from `scripts/` to a known path)

### 9.4 sops-nix configuration during first boot

On first boot, `install-config.yaml` is only encrypted for the operator and
Yubikey install keys — not the host SSH key. sops-nix is configured to use
the SSH host key (`sops.age.sshKeyPaths`), so it will fail on first boot
before enrollment.

`/run/age-install-key` does not exist on the installed system — `/run` is a
fresh tmpfs on every boot and the installer's `yk-unwrap.service` only runs
on the installer ISO.

The enrollment service handles this by running `yk-unwrap` itself on first
boot, writing the install age key to `/run/age-install-key`. sops-nix is
configured with both key sources:

```nix
sops.age.sshKeyPaths = [ "/etc/ssh/ssh_host_ed25519_key" ];
sops.age.keyFile     = "/run/age-install-key";
```

sops-nix tries all configured keys; whichever one matches a recipient in
`install-config.yaml` succeeds. On first boot the install age key matches;
on subsequent boots the SSH host key matches.

The enrollment service ordering:

```
killy-enroll-host-key.service
  After=  sshd-keygen.service
  Before= sops-install-secrets.service
  Wants=  sops-install-secrets.service
```

First boot sequence:

1. `sshd-keygen` generates `/etc/ssh/ssh_host_ed25519_key`
2. Enrollment service checks: can host key decrypt `install-config.yaml`? No.
3. Enrollment service runs `yk-unwrap` → writes `/run/age-install-key`
4. Enrollment service updates `.sops.yaml`, runs `sops updatekeys`
5. `sops-install-secrets` activates — uses `/run/age-install-key` (host key
   is now also a recipient but either key works)

Subsequent boots:

1. `sshd-keygen` generates `/etc/ssh/ssh_host_ed25519_key` (already exists, no-op)
2. Enrollment service checks: can host key decrypt `install-config.yaml`? Yes → exit immediately.
3. `sops-install-secrets` activates using SSH host key
